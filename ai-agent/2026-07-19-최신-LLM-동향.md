### Context
* 2026-07-19 기준 Google 검색(SerpApi)을 통해 최신 LLM 관련 정보를 수집함
* 사용자가 최신 LLM 정보를 검색하여 기록(TIL)으로 남길 것을 요청함
* 주요 관심사는 LLM 기본 개념, 웹 검색 연동, RAG, 오픈소스/로컬 LLM, 지식 저장 메커니즘임

### Core
* LLM 정의: 방대한 데이터로 사전학습된 초대형 딥러닝 모델 (AWS, ServiceNow)
  * 기본 구조는 Transformer 기반의 인코더-디코더 신경망 세트
  * 셀프 어텐션(self-attention) 메커니즘으로 텍스트 이해 및 생성 수행
* 최신 웹 검색 연동
  * LLM은 인터넷 검색 결과를 어텐션 메커니즘을 통해 이해하고 활용 (brunch.co.kr, 2025-08-02)
  * 인터넷 + Thinking 조합으로 리포트 생성 가능 (Threads, 2025-03-03)
* RAG (Retrieval-Augmented Generation)
  * LLM의 환각(Hallucination) 및 정보 부족 문제를 해결하는 지능형 검색 시스템 (YouTube Codedeck)
  * 벡터 데이터베이스 + LangChain + LLM 조합으로 개인 데이터 의미론적 검색 구현 (LinkedIn)
* 오픈소스 / 로컬 LLM
  * LM Studio, Anything LLM 등 로컬에서 실행 가능한 도구 활용
  * 로컬 LLM에 웹 검색 및 파일 업로드(RAG) 기능을 결합하는 사례 증가
* 지식 저장 및 검색 메커니즘
  * LLM은 지식을 가중치로 압축(손실 압축)하여 저장 (Reddit r/LocalLLaMA)
  * 간단한 선형 함수로 관계 정보를 디코딩하며, 지식 유형에 따라 디코딩 함수가 다름 (gnict.org, 2024-04-03)

### Insight
* LLM 단독으로는 최신 정보 반영에 한계가 있어 웹 검색/외부 도구 연동이 표준 패턴으로 정착
* RAG는 환각 문제 해결과 최신성 보완을 위한 실무 핵심 기술
* 로컬 LLM 생태계(LM Studio, Anything LLM) 확대로 프라이버시 중심 활용 증가
* 지식이 가중치에 분산 저장된다는 점은 해석 가능성(interpretability) 연구의 중요성 시사
* 검색 결과 기준 2026-07-19 시점에서도 개념 설명 위주 콘텐츠가 다수이며, 실시간 모델 벤치마크는 별도 추적 필요
