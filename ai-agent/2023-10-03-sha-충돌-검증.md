### Context
SHA(Secure Hash Algorithm)는 데이터의 무결성을 검증하기 위해 사용됩니다. 데이터나 파일의 변경 여부를 확인하기 위해 SHA 해시를 생성하고 저장합니다. 하지만 동일한 해시 값을 가진 두 개의 다른 입력값이 존재할 경우를 SHA 충돌이라고 합니다. 본 문서에서는 SHA 충돌 검증 방법에 대해 설명합니다.

### Core
* SHA-1 또는 SHA-256 해시 함수를 사용하여 파일의 해시 값을 생성합니다.  
    ```python
    import hashlib
    
def generate_sha256(file_path):
    hash_sha256 = hashlib.sha256()
    with open(file_path, 'rb') as f:
        for byte_block in iter(lambda: f.read(4096), b''):
            hash_sha256.update(byte_block)
    return hash_sha256.hexdigest()
    ```  
* 두 개의 파일에 대해 각각 SHA 해시 값을 생성한 후 비교합니다.  
    ```python
    file1_hash = generate_sha256('file1.txt')
    file2_hash = generate_sha256('file2.txt')
    if file1_hash == file2_hash:
        print('충돌 발생!')
    else:
        print('충돌 없음')
    ``` 

### Insight
SHA 충돌을 검증하는 것은 데이터 무결성을 유지하고, 동일한 해시 값이 생성될 경우 문제가 발생할 수 있는 위험성을 관리하는 데 중요합니다. SHA-1은 끊임없는 보안 취약점으로 인해 사용이 줄어들고 있으며, SHA-256 같은 강력한 해시 함수를 사용하는 것이 바람직합니다.
