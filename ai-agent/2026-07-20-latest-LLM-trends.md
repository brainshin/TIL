### Context
* 최신 LLM 생태계와 웹 검색 연동, 로컬 LLM 활용, 작동 원리에 대한 이해가 필요함
* 2026-07-20 Google 검색(SerpApi)을 통해 수집한 최신 자료를 바탕으로 정리
* 최신 정보는 빠르게 변화하므로 출처와 날짜를 함께 기록

### Core
* LLM 정의: 대규모 언어 모델(Large Language Model)은 방대한 텍스트 데이터를 학습해 언어를 이해하고 생성하는 딥러닝 모델
* 아키텍처: 트랜스포머(Transformer) 기반의 인코더-디코더 신경망 구조 사용
* 웹 검색 연동: LLM이 인터넷 검색 도구를 호출해 최신 정보를 가져오고, Thinking(추론)과 결합해 환각(Hallucination)을 줄임
* 로컬 LLM: Ollama, LM Studio 등을 통해 모델을 로컬에서 실행하고 도구 호출(Tool Calling)로 웹 검색 가능
* RAG(검색 증강 생성): 외부 지식 소스에서 관련 문서를 검색해 답변에 반영하는 실전 기법
* 주요 학습/구현 자료
  * LLM 바닥부터 만들기 (홍정모) - 토크나이징, 데이터로더, 한국어 학습 과정 포함
  * Cloudflare / ServiceNow 의 LLM 개념 정리 문서
  * 브런치 'LLM은 어떻게 최신 웹 검색 결과를 이해할까?' (2025-08-02)

### Insight
* LLM을 단순 압축 모델이 아닌 '검색 + 추론' 에이전트로 보는 관점이 최신 트렌드
* 환각 문제는 웹 검색 연동과 출처 제시로 상당 부분 완화 가능
* 로컬 LLM 생태계(Ollama, LM Studio)의 성장으로 프라이버시와 비용 측면에서 온프레미 활용이 확대됨
* 실무에서는 RAG와 멀티모달 처리가 핵심 과제로 부상
* 검색 결과는 시간 의존적이므로 정기적인 갱신과 출처 관리가 필요
