### Context
최근 AI 에이전트 구축 시 외부 도구(Tools)와의 연동이 필수적입니다. Claude가 제시한 모델 컨텍스트 프로토콜(MCP, Model Context Protocol)은 다양한 시스템을 표준화된 방식으로 AI 에이전트에 연결하는 표준입니다. n8n은 강력한 워크플로우 자동화 도구로서, MCP 서버와 연동하여 레거시 시스템을 에이전트의 도구로 쉽게 편입시킬 수 있는 강력한 환경을 제공합니다.

### Core
n8n에서 MCP를 사용하여 시스템을 연동하는 핵심 로직은 'MCP Tool' 노드를 활용하는 것입니다. 이를 위해 다음과 같은 구성을 따릅니다.

* **MCP 서버 설정**: n8n 환경 내에서 실행되는 MCP 서버 또는 외부 MCP 서버 엔드포인트를 연결합니다.
* **AI 에이전트 노드 배치**: `AI Agent` 노드를 생성하고, 하위 도구로 `MCP Tool`을 추가합니다.
* **설정 예시**:
```json
// n8n AI Agent 설정 개념
{
  "node": "AI Agent",
  "parameters": {
    "tools": [
      {
        "node": "MCP Tool",
        "parameters": {
          "serverUrl": "http://localhost:3000/mcp", 
          "toolName": "legacy-db-query"
        }
      }
    ]
  }
}
```
* **워크플로우 흐름**:
  1. 사용자가 질문 입력.
  2. AI 에이전트가 질문 의도를 파악하여 연결된 MCP 툴이 필요한지 판단.
  3. n8n이 MCP 서버와 통신하여 데이터 조회/수행.
  4. 결과를 AI가 해석하여 사용자에게 응답.

### Insight
n8n에서 MCP를 활용하는 것은 레거시 자동화의 판도를 바꾸는 작업입니다. 기존의 노드 방식(개별 API 노드 연결)이 특정 시스템마다 설정을 직접 해야 했다면, MCP를 통하면 표준 규격에 맞는 서버만 연동함으로써 즉시 수백 개 이상의 도구(Tools)를 에이전트가 활용하도록 확장할 수 있습니다. 특히 인증(Authentication) 처리를 n8n 수준에서 관리할 수 있어 보안 측면에서도 유리합니다.
