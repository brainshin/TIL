### Context
SHA-1 해시 알고리즘은 이제는 안전하지 않다는 점이 여러 연구를 통해 확인되었고, 이로 인해 SHA-1의 충돌 공격이 현실적으로 가능하다는 사실이 밝혀졌다. 많은 서비스와 시스템이 여전히 SHA-1을 사용하는 상황에서 이러한 공격이 발생할 가능성은 매우 크다. 따라서, SHA-1의 충돌 검증 및 이를 회피하는 방법이 필요하다.

### Core
SHA-1 충돌 공격을 검증하기 위해 다음의 과정을 따른다:
* SHA-1 해시 함수의 구조 이해
* 두 개의 입력 데이터(예: 파일)를 준비하여 해시 값을 생성
* 해시 값이 동일한지 검증
* 충돌이 발생하는지 확인하기 위해 다수의 입력을 시도한다.

아래는 Python에서 SHA-1 해시를 생성하고 충돌을 검증하는 간단한 코드 예시이다:

```python
import hashlib

def check_sha1_collision(data1, data2):
    sha1_hash1 = hashlib.sha1(data1).hexdigest()
    sha1_hash2 = hashlib.sha1(data2).hexdigest()
    return sha1_hash1 == sha1_hash2

# 테스트용 데이터
data1 = b"This is a test data."

data2 = b"This is a different data."

# 충돌 검증
collision = check_sha1_collision(data1, data2)
print(f"Collision detected: {collision}")
```
