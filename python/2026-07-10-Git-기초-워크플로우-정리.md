### Context
Git은 버전 관리 시스템(Version Control System)으로, 프로젝트의 변경 사항을 추적하고 협업하는 데 필수적입니다. 로컬 저장소 초기화부터 브랜치 생성, 변경 사항 반영, 그리고 원격 저장소로 업로드하는 표준 워크플로우를 이해하는 것이 중요합니다.

### Core
Git의 기본적인 워크플로우는 다음과 같습니다.

*   **초기화 (Initialize)**: 프로젝트 폴더를 Git 저장소로 설정합니다.
    ```bash
    git init
    ```

*   **브랜치 변경 (Branching)**: 새로운 브랜치를 생성하고 이동합니다.
    ```bash
    # 생성 및 즉시 이동
    git checkout -b <브랜치명>
    # 또는 최신 버전의 git에서는
    git switch -c <브랜치명>
    ```

*   **커밋 (Commit)**: 변경된 파일을 스테이징 영역에 추가하고 로컬 저장소에 기록합니다.
    ```bash
    git add .
    git commit -m "커밋 메시지 작성"
    ```

*   **푸시 (Push)**: 로컬의 커밋 내역을 원격 저장소(GitHub)로 전송합니다.
    ```bash
    git push origin <브랜치명>
    ```

*   **간편화된 처리 방법**: 위 과정을 한 번에 수행하는 간편한 방법은 없습니다. 하지만 터미널에서 **Shell Aliases**를 사용하거나, **GitHub Desktop**과 같은 GUI 클라이언트를 사용하면 매번 명령어를 입력하는 번거로움을 줄일 수 있습니다.
    *   **Alias 예시 (.gitconfig)**:
        ```bash
        [alias]
        save = !git add . && git commit -m
        ```
        이렇게 설정하면 `git save "메시지"` 한 번으로 `add`와 `commit`을 동시에 실행할 수 있습니다.

### Insight
Git은 CLI 환경에서 다루는 것이 가장 강력하지만, 입문자에게는 명령어가 생소할 수 있습니다. `alias` 기능을 활용하여 자주 사용하는 명령어 조합을 단축어로 만들어 사용하면 작업 효율을 크게 높일 수 있습니다. 또한 최근에는 VS Code의 내장 Git 기능을 활용하는 것도 좋은 방법입니다.
