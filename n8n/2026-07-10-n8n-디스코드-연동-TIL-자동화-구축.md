### Context
디스코드 메시지를 수신했을 때 이를 자동으로 감지하여 TIL(Today I Learned) 형식으로 변환하고, GitHub 리포지토리에 푸시(Push)한 뒤 README.md의 인덱스를 업데이트하여 최종 완료 알림을 보내는 자동화 워크플로우를 구축하고자 합니다. 이 과정은 수동으로 진행하던 문서 기록 및 관리 작업을 n8n을 통해 자동화하여 생산성을 높이기 위한 목적입니다.

### Core
n8n 워크플로우는 다음 노드들을 연결하여 구성합니다.

* **Discord Trigger**: 메시지 이벤트를 수신합니다.
* **AI Agent (OpenAI/Anthropic)**: 수신된 메시지를 TIL 마크다운 형식으로 변환합니다.
* **GitHub Node**: 변환된 마크다운 파일을 특정 경로에 푸시합니다.
* **GitHub Node (Update)**: README.md 파일을 조회하여 최신 TIL 파일 링크를 추가합니다.
* **Discord Webhook**: 작업 완료 알림을 전송합니다.

```json
{
  "nodes": [
    {
      "parameters": {
        "event": "messageCreate"
      },
      "type": "n8n-nodes-base.discordTrigger",
      "name": "Discord Trigger"
    },
    {
      "parameters": {
        "promptType": "define",
        "text": "다음 내용을 바탕으로 TIL 마크다운 형식을 작성하세요: {{ $json.message }}"
      },
      "type": "n8n-nodes-base.aiAgent",
      "name": "TIL Converter"
    },
    {
      "parameters": {
        "operation": "createFile",
        "owner": "username",
        "repo": "til-repo",
        "path": "docs/{{ $now.toFormat('yyyy-MM-dd') }}-title.md",
        "content": "{{ $json.output }}"
      },
      "type": "n8n-nodes-base.github",
      "name": "GitHub Push"
    }
  ]
}
```

### Insight
n8n을 처음 학습할 때 가장 중요한 것은 각 노드의 입력(Input) 데이터 구조와 출력(Output) 데이터 스키마를 정확히 파악하는 것입니다. 이 두 가지만 이해한다면 복잡한 워크플로우라도 노드 간 데이터를 어떻게 전달하고 변환할지 쉽게 설계할 수 있습니다. 반복적인 수동 작업을 워크플로우로 자동화하는 경험은 기술적 역량을 빠르게 향상시키는 데 매우 효과적입니다.
