### Context
n8n에서 하나의 노드(Node)가 실행될 때, 이전 노드로부터 전달된 데이터(Input Items)가 여러 개이거나, 해당 노드 자체가 여러 개의 아이템을 생성하는 로직(예: Split In Batches, HTTP Request 결과의 배열 처리 등)을 가지고 있을 경우, n8n은 이를 **아이템 단위로 반복 실행**합니다. 사용자가 기대한 것보다 많은 결과가 출력되는 이유는 노드의 출력 방식이 각 입력 아이템에 대해 1:1로 대응하여 결과를 생성하기 때문입니다.

### Core
n8n의 데이터 처리 방식은 기본적으로 각 아이템(Item)을 독립적으로 취급합니다. 만약 특정 노드의 결과를 하나로 합치고 싶다면 `Item Lists` 노드를 활용해야 합니다.

* **Split In Batches (데이터 분할)**: 데이터를 작은 묶음으로 나누어 처리할 때 사용하며, 각 묶음마다 별도의 출력을 생성합니다.
* **Aggregate (데이터 병합)**: 여러 개의 아이템을 하나의 리스트 혹은 단일 오브젝트로 합칠 때 사용합니다.
    * 노드 이름: `Item Lists`
    * 작업 유형: `Aggregate Items`
    * 옵션 설정: `Aggregate All Items`를 선택하면 모든 입력을 하나의 배열로 병합합니다.

```json
{
  "nodes": [
    {
      "parameters": {
        "operation": "aggregateItems",
        "aggregate": "aggregateAllItemData"
      },
      "type": "n8n-nodes-base.itemLists",
      "typeVersion": 3,
      "position": [400, 300]
    }
  ]
}
```

### Insight
n8n은 기본적으로 **Array of Items** 구조를 기반으로 작동합니다. 따라서 이전 노드에서 3개의 아이템이 넘어왔다면, 후속 노드는 특별한 제어(Aggregation)가 없는 한 3번 실행되거나 3개의 출력 결과를 생성하는 것이 정상적인 동작(Expected Behavior)입니다. 이를 이해하는 것은 복잡한 워크플로우에서 데이터 누락이나 중복을 방지하는 핵심입니다.
