### Context
사용자가 현재 날씨와 더위에 대해 언급함에 따라, 외부 날씨 데이터를 실시간으로 가져와 처리하는 워크플로우를 자동화하고자 함. 특히 n8n을 사용하여 외부 API(예: OpenWeatherMap)와 연동하여 현재 온도를 확인하고, 특정 임계값 이상의 온도일 경우 알림을 보내는 시나리오를 구축하고자 함.

### Core
n8n에서 날씨 데이터를 가져오기 위한 기본적인 HTTP Request 노드 설정 예시:

```json
{
  "nodes": [
    {
      "parameters": {
        "method": "GET",
        "url": "https://api.openweathermap.org/data/2.5/weather",
        "sendQuery": true,
        "queryParameters": {
          "parameters": [
            {
              "name": "q",
              "value": "Seoul"
            },
            {
              "name": "appid",
              "value": "YOUR_OPENWEATHERMAP_API_KEY"
            },
            {
              "name": "units",
              "value": "metric"
            }
          ]
        }
      },
      "type": "n8n-nodes-base.httpRequest",
      "typeVersion": 4.1,
      "position": [450, 300],
      "id": "weather-node"
    }
  ]
}
```

### Insight
* 날씨 데이터 연동 시 OpenWeatherMap과 같은 API를 사용하면 단순 온도뿐만 아니라 습도, 체감 온도 등 다양한 기상 정보를 파라미터로 받을 수 있음.
* '더위'에 대한 판단은 단순 기온뿐만 아니라 습도(Humidity)를 고려한 불쾌지수 계산식을 적용하면 더욱 정교한 알림 시스템 설계가 가능함.
* 외부 API 호출 시 API Key 보안을 위해 n8n의 Credentials 기능을 사용하는 것을 권장함.
