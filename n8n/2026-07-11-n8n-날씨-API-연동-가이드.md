### Context
n8n과 같은 자동화 도구에서 날씨 정보를 활용하기 위해 OpenWeatherMap API를 연동하는 방법을 정리합니다. 사용자 위치의 실시간 기상 정보를 자동으로 수집하여 알림이나 데이터 분석에 활용하는 시나리오를 위한 기술적 가이드입니다.

### Core
n8n에서 날씨 API를 연동하는 기본적인 워크플로우 구성 방법입니다.

* HTTP Request 노드를 사용하여 OpenWeatherMap API에 요청을 보냅니다.
* API Key를 환경 변수나 Credential로 안전하게 관리합니다.
* JSON 응답을 Parse하여 필요한 기온, 습도, 날씨 상태를 추출합니다.

```json
{
  "nodes": [
    {
      "parameters": {
        "url": "https://api.openweathermap.org/data/2.5/weather",
        "method": "GET",
        "queryParameters": {
          "parameters": [
            { "name": "q", "value": "Seoul" },
            { "name": "appid", "value": "YOUR_API_KEY" },
            { "name": "units", "value": "metric" }
          ]
        }
      },
      "type": "n8n-nodes-base.httpRequest",
      "name": "Get Weather Data"
    }
  ]
}
```

### Insight
* API 호출 시 단위(`units=metric`)를 설정하여 섭씨 온도를 받아오는 것이 중요합니다.
* 무료 티어의 경우 API 요청 제한이 있으므로, 에이전트 구축 시 요청 간격(Cron 노드 활용)을 최적화해야 합니다.
* 기상 데이터의 변화에 따른 조건부 분기(If 노드)를 추가하여 비 예보 시 자동 알림을 발송하는 워크플로우로 확장이 용이합니다.
