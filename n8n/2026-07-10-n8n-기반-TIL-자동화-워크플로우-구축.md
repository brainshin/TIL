### Context
디스코드 메시지를 통해 수신된 학습 내용을 자동으로 TIL(Today I Learned) 형식으로 변환하여 GitHub 저장소에 기록하고, README.md 파일에 인덱스를 업데이트하는 자동화 워크플로우를 구축함.

### Core
n8n 워크플로우의 핵심 로직은 다음과 같이 구성됨:

*   **Discord Trigger**: 디스코드 채널의 메시지 수신을 감지하여 워크플로우를 트리거링함.
*   **AI Agent Node**: 수신된 텍스트를 입력받아 지정된 TIL 포맷에 맞춰 본문을 생성.
*   **GitHub Node**: 생성된 마크다운 파일을 지정된 저장소에 커밋 및 푸시.
*   **Update README.md**: 추가된 TIL 파일을 인덱싱하여 README에 링크를 삽입.
*   **Discord Send**: 모든 프로세스 완료 후 디스코드로 결과 메시지를 발송.

```json
{
  "trigger": "Discord Trigger",
  "nodes": [
    "AI Agent (LLM)",
    "GitHub (Commit & Push)",
    "GitHub (Update README)",
    "Discord Send"
  ]
}
```

### Insight
n8n 자동화 설계의 핵심은 **데이터의 흐름(Data Flow)**을 이해하는 것임. 각 노드가 가진 **입력(Input) 스키마**와 **출력(Output) 데이터** 구조만 명확히 파악하면, 복잡한 자동화도 간결하게 구현할 수 있음. 노드 간 데이터 전달(JSON 객체 매핑)을 최적화하는 것이 워크플로우 안정성을 높이는 비결임.
