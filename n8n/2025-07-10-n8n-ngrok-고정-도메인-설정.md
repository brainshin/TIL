### Context
n8n을 로컬 환경(예: Docker)에서 실행할 때, 웹훅(Webhook) 트리거를 사용하여 외부 서비스(Slack, GitHub, Notion 등)와 연동하려면 해당 서버가 외부에서 접근 가능한 공인 주소(Public URL)를 가져야 합니다. 하지만 로컬 서버는 기본적으로 내부 IP 주소만 가지므로, 이를 외부로 안전하게 노출하고 고정된 도메인을 사용하기 위해 ngrok을 활용합니다.

### Core
ngrok의 무료 플랜에서도 제공하는 고정 도메인 기능을 활용하여 n8n을 외부와 연결하는 방법입니다.

* **ngrok 도메인 생성**: ngrok 대시보드(ngrok.com)에 가입한 후 'Domains' 메뉴에서 무료 고정 도메인을 생성합니다 (예: `your-name.ngrok-free.app`).
* **실행 명령어**: 다음 명령어를 통해 로컬의 n8n 포트(기본 5678)를 ngrok 고정 도메인으로 포워딩합니다.
```bash
# 기본 포워딩 명령어
ngrok http 5678 --domain=your-name.ngrok-free.app
```
* **n8n 환경 변수 설정**: 외부에서 웹훅 요청을 올바르게 받기 위해 n8n의 `WEBHOOK_URL` 환경 변수를 설정해야 합니다.
```bash
# Docker 실행 예시
docker run -d --name n8n -p 5678:5678 \
  -e WEBHOOK_URL=https://your-name.ngrok-free.app/ \
  n8nio/n8n
```

### Insight
ngrok을 통한 터널링은 별도의 복잡한 공유기 포트 포워딩 설정 없이도 즉시 HTTPS 보안 환경을 구축할 수 있는 가장 효율적인 방법입니다. 특히 고정 도메인을 설정하면 매번 ngrok 주소가 변경되어 n8n의 `WEBHOOK_URL`을 수정해야 하는 번거로움을 방지할 수 있습니다. 

단, 개발 및 테스트 용도로는 매우 훌륭하지만, 서비스의 안정성과 가용성을 위해 운영 환경(Production)으로 이전할 때는 클라우드 인프라(AWS, GCP 등)를 사용하는 것을 권장합니다.
