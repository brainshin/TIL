### Context
n8n 워크플로우의 소스 코드를 수정할 때, 변경 사항이 예상대로 작동하는지 확인하는 것이 중요하다. 워크플로우의 소스 수정 후 검증을 통해 오류를 사전에 발견하고, 시스템 전반에 미치는 영향을 최소화할 수 있다.

### Core
* n8n의 실행 모드를 `Test`로 설정하여 변경된 워크플로우를 테스트한다. 
* 각 노드의 상태를 확인해 정상적으로 작동하는지 검증한다.
* `Webhook` 노드의 경우, 실제 트리거를 보내서 응답을 확인하는 테스트를 진행한다.
* 각종 환경 변수나 API 키가 정상적으로 설정되었는지 검토한다.
* `Execution` 메뉴를 통해 실행 로그를 분석하여 오류를 판별하고, 필요한 경우 설정을 수정한다. 

### Insight
워크플로우의 소스 수정을 검증하는 과정은 간단하지만, 필수적인 단계이다. 특히, 여러 단계로 구성된 복잡한 워크플로우는 각 단계를 독립적으로 테스트하여 문제를 조기에 발견해 수정할 수 있다. n8n을 효과적으로 활용하기 위해 이러한 검증 절차는 반드시 수행해야 한다.
