### Context
일상적인 날씨 문의는 사용자의 현재 위치 정보가 포함되지 않을 경우 정확한 예측이 불가능합니다. 특히 '내일은 비가 올까?'라는 질문은 날씨 정보를 제공하는 API(예: OpenWeatherMap, 기상청 API)와의 연동이 필수적이며, LLM이 단순히 자연어 처리만으로는 실시간 기상 데이터를 파악하는 데 한계가 있음을 보여주는 사례입니다.

### Core
n8n을 활용하여 날씨 정보를 자동화하고 싶다면 다음과 같은 워크플로우를 설계할 수 있습니다. 

* **HTTP Request Node**: OpenWeatherMap API(https://api.openweathermap.org/)를 호출하여 JSON 데이터를 수신합니다.
* **Set Node**: API 응답값에서 `weather` 배열 내 `main` 또는 `description` 필드를 추출합니다.
* **If Node**: 날씨 상태값에 'Rain' 혹은 'Drizzle'이 포함되는지 조건문을 설정합니다.
* **AI Agent / Chat Trigger**: 사용자의 메시지를 받아 위 노드들을 통해 얻은 결과를 바탕으로 답변을 구성합니다.

```json
{
  "nodes": [
    {
      "parameters": {
        "url": "https://api.openweathermap.org/data/2.5/forecast",
        "options": {
          "queryParameters": {
            "parameters": [
              { "name": "q", "value": "Seoul" },
              { "name": "appid", "value": "YOUR_API_KEY" }
            ]
          }
        }
      },
      "type": "n8n-nodes-base.httpRequest"
    }
  ]
}
```

### Insight
일반적인 LLM 에이전트 구축 시 외부 서비스와의 연동(Tool Calling)은 필수적입니다. 날씨와 같이 시시각각 변하는 데이터는 내부 학습된 지식(Training Data)으로 답할 수 없으며, 반드시 실시간 데이터 소스(Weather API)와 연동해야 합니다. 사용자 질문에 대응하기 위해서는 위치 기반의 API 호출 구조를 반드시 갖추어야 정확한 정보 제공이 가능합니다.
